Full Ethernet Frame

1. Data-Link Layer (Ethernet Frame – With 802.1Q VLAN Tag)

Ethernet Header

  • Preamble (7 bytes) – Synchronization for receiver
  • Start Frame Delimiter (SFD) (1 byte) – Indicates start of frame
  • Destination Mac Address (6 bytes)
  • Source Mac Address 1 (6 bytes)
  • 802.1Q VLAN Tag (4 bytes)
    • Tag Protocol Identifier (TPID) (2 bytes) – Set to 0x8100 for VLAN-tagged frames
    • Priority Code Point (PCP) (3 bits) – QoS priority (0-7)
    • Drop Eligible Indicator (DEI) (1 bit) – Indicates if frame can be dropped
    • VLAN Identifier (VID) (12 bits) – VLAN number (0-4095)
  • Type - Length (2 bytes)
    • If value ≥ 0x0600, it’s an EtherType (e.g., 0x0800 for IPv4)
    • If value ≤ 1500, it represents payload length (IEEE 802.3 format)

Payload (46–1500 bytes)

  • Contains the encapsulated Network Layer (Layer 3) packet

Frame Check Sequence (FCS) (4 bytes)

  • Cyclic Redundancy Check (CRC) (4 bytes) – Error detection

2. Network Layer (IPv4 Packet)

IPv4 Header (Min 20 bytes, Max 60 bytes)

  • Version (4 bits) – IPv4 (4) or IPv6 (6)
  • Internet Header Length (IHL) (4 bits) – Length of the header in 32-bit words (min 5 = 20 bytes)
  • Differentiated Services (DS) Field (1 byte)
    • Differentiated Services Code Point (DSCP) (6 bits) – QoS classification
    • Explicit Congestion Notification (ECN) (2 bits) – Congestion control
  • Total Length (2 bytes) – Total size of packet (header + payload), max 65,535 bytes
  • Identification (2 bytes) – ID for fragmentation & reassembly
  • Flags (3 bits)
    • Reserved Bit (1 bit) – Always 0
    • Don’t Fragment (DF) (1 bit) – Prevents fragmentation
    • More Fragments (MF) (1 bit) – Indicates more fragments follow
  • Fragment Offset (13 bits) – Position of fragment in original packet
  • Time To Live (TTL) (1 byte) – Packet lifetime (decremented each hop)
  • Protocol (1 byte) – Encapsulated Layer 4 protocol (e.g., TCP = 6, UDP = 17)
  • Header Checksum (2 bytes) – Error checking for header
  • Source IP Address (4 bytes)
  • Destination IP Address (4 bytes)
  • Options (0–40 bytes, optional) – Used for special routing, security, timestamping

Payload (Variable)

  • Encapsulated Transport Layer (Layer 4) segment

3. Transport Layer (TCP/UDP Segments)

TCP Segment Header (Min 20 bytes, Max 60 bytes)

  • Source Port (2 bytes) – Sender’s port number
  • Destination Port (2 bytes) – Receiver’s port number
  • Sequence Number (4 bytes) – Position of first byte in data stream
  • Acknowledgment Number (4 bytes) – Next expected byte from sender
  • Data Offset (4 bits) – TCP header size in 32-bit words (min 5 = 20 bytes)
  • Reserved (3 bits) – Must be 0
  • Control Flags (9 bits)
    • NS (1 bit) – ECN nonce
    • CWR (1 bit) – Congestion window reduced
    • ECE (1 bit) – Explicit congestion notification
    • URG (1 bit) – Urgent pointer field is valid
    • ACK (1 bit) – Acknowledgment field is valid
    • PSH (1 bit) – Push function (deliver data immediately)
    • RST (1 bit) – Reset connection
    • SYN (1 bit) – Synchronize sequence numbers (used in handshake)
    • FIN (1 bit) – Terminate connection
  • Window Size (2 bytes) – Number of bytes receiver is willing to accept
  • Checksum (2 bytes) – Error-checking for header and payload
  • Urgent Pointer (2 bytes) – If URG flag is set, indicates urgent data offset
  • Options (0–40 bytes, optional)
    • Common options: MSS (Max Segment Size), Window Scale, Timestamps
  • Padding (Variable) – Ensures header ends at a 32-bit boundary

TCP Payload (Variable)

  • Application-layer data

UDP Segment Header (8 bytes total)

  • Source Port (2 bytes)
  • Destination Port (2 bytes)
  • Length (2 bytes) – Header + data size
  • Checksum (2 bytes) – Error detection

UDP Payload (Variable)

  • Application-layer data
